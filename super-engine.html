Contrôleur non détecté ?
Chrome utilise le MIDI système Windows (WinMM). Certains contrôleurs (Novation, Akai…) installent un driver propriétaire invisible à Chrome. Solution : passer par loopMIDI + MIDI-OX.
MARCHE À SUIVRE
1. Installer loopMIDI — tobias-erichsen.de
2. Installer MIDI-OX — midiox.com
2. Ouvrir MIDI-OX → menu Options → MIDI Devices
3. Colonne gauche MIDI Inputs → sélectionner ton contrôleur
4. Colonne droite MIDI Outputs → sélectionner loopMIDI
5. Menu Actions → Port Routings → New Simple Port Route
    Input : contrôleur · Output : loopMIDI → Route
6. Ici : sélectionner loopMIDI dans le sélecteur APPAREIL
MIDI-OX doit rester ouvert en arrière-plan pendant la session.

Sortie vidéo — OBS
Super Engine VJ tourne dans le navigateur. Pour envoyer l'image vers un logiciel vidéo ou un réseau, utilisez OBS Studio comme pont.
① CAMÉRA VIRTUELLE OBS
Principe : OBS capture la fenêtre du navigateur et expose une caméra virtuelle utilisable dans n'importe quel logiciel (MadMapper, Resolume, Zoom, vMix…).

1. Installer OBS Studio — obsproject.com
2. Passer Super Engine en plein écran (F) et masquer le panneau (H)
3. Dans OBS → Sources → Capture de fenêtre → sélectionner le navigateur
4. Régler la résolution de la scène OBS pour qu'elle corresponde au canvas
5. Démarrer la caméra virtuelle (menu Outils → Démarrer la caméra virtuelle)
La caméra virtuelle est incluse dans OBS depuis v26 — aucun plugin requis.
② NDI — RÉSEAU LOCAL (obs-ndi)
Principe : NDI envoie le flux vidéo sur le réseau local sans câble. Tout logiciel compatible NDI (Resolume, Arena, vMix, TouchDesigner…) peut recevoir le flux.

1. Installer NDI Tools — ndi.video/tools (inclut le runtime NDI)
2. Installer le plugin obs-ndi — github.com/obs-ndi/obs-ndi
3. Redémarrer OBS — un menu Outils → NDI Output Settings apparaît
4. Activer Main Output et nommer la source (ex : SuperEngineVJ)
5. Dans le logiciel destinataire → ajouter une source NDI → sélectionner SuperEngineVJ

Tous les appareils doivent être sur le même réseau local. Le flux est en temps réel, faible latence.
OBS → obsproject.com  ·  obs-ndi → github.com/obs-ndi/obs-ndi  ·  NDI Tools → ndi.video

Les 8 moteurs de simulation

FLUID Eulérien · Grille

Simulation de fluide incompressible sur une grille fixe. Le champ de vélocité est mis à jour par advection, diffusion et projection (divergence nulle). Les particules sont de simples traceurs qui suivent le flux — elles ne sont pas le fluide.

La souris injecte de la vélocité dans les cellules voisines. Le post-process WebGL applique un warp (déformation de l'image) synchronisé sur les pulses.

Navier-Stokes simplifiéAdvectionWebGL warp
VORTEX Lagrangien · Tourbillons ponctuels

Chaque tourbillon (vortex de Rankine) génère un champ de rotation à distance. Les tourbillons s'influencent mutuellement et se déplacent selon la somme des champs. Avec 3 tourbillons ou plus, le système devient chaotique.

Les particules légères tracent les lignes de courant. La souris peut injecter de nouveaux tourbillons positifs ou négatifs (sens horaire / anti-horaire).

Mécanique des fluides 2DChaos déterministeO(n²)
N-BODY Gravitation newtonienne

Les corps massifs s'attirent selon F = G·m₁·m₂/r². Un softening évite les singularités à courte distance. Phénomènes observables : orbites stables, éjections hyperboliques, captures, fusions.

Les particules légères (specks) tracent le champ gravitationnel sans masse propre. La souris attire ou repousse les corps.

Problème à N corpsO(n²)Intégration Euler
SPH Smoothed Particle Hydrodynamics

Contrairement au Fluid, les particules sont le fluide. Chaque particule calcule sa densité locale (kernel Poly6) et sa pression. Les forces de pression (gradient Spiky) et de viscosité (laplacien) agissent entre voisins dans un rayon h.

Simule des comportements eau : gouttelettes, splash, tension de surface implicite. Plus lent que l'Eulérien mais plus réaliste pour les fluides libres.

LagrangienKernel Poly6 / SpikyGrille spatiale
BOIDS Intelligence collective émergente

Algorithme de Craig Reynolds (1986). Chaque boid applique trois règles locales : Séparation (éviter les collisions), Alignement (adopter la direction du voisinage), Cohésion (se rapprocher du centre du groupe).

Aucun chef, aucune coordination globale — le banc, la nuée ou le troupeau émerge spontanément des interactions locales.

Vie artificielleComportement émergentO(n²) naïf
PHYSARUM Moisissure visqueuse · Phéromones

Inspire du Physarum polycephalum, un organisme unicellulaire réel capable de résoudre des labyrinthes. Des milliers d'agents déposent des phéromones et orientent leurs capteurs vers les concentrations les plus fortes — les traces se renforcent mutuellement.

Résultat : des réseaux tubulaires optimaux apparaissent spontanément, similaires aux réseaux de transport biologiques. Proche des fourmis (ACO) mais l'organisme est unicellulaire.

StigmergiePhéromonesDiffusion · Décroissance60 000 agents
LORENZ Attracteurs étranges · Chaos

Trajectoires de systèmes dynamiques chaotiques projetées en 2D. Quatre attracteurs disponibles : Lorenz (papillon, 1963), Rössler, Clifford (attracteur plan), Thomas (cyclique).

Deux trajectoires initialement voisines divergent exponentiellement (exposant de Lyapunov positif) — c'est l'effet papillon. Les trajectoires restent pourtant confinées dans le volume de l'attracteur.

Systèmes dynamiquesChaos déterministeProjection 3D→2D
REACT Réaction-Diffusion · Gray-Scott · GPU

Deux substances chimiques U et V réagissent (V consomme U, se reproduit, se dégrade) et diffusent à des vitesses différentes. Le modèle Gray-Scott : dU/dt = Dᵤ∇²U − UV² + f(1−U) · dV/dt = D_v∇²V + UV² − (f+k)V.

Selon les paramètres f (feed) et k (kill) : spots, labyrinthes, ondes spirales, mitose cellulaire. Calculé entièrement sur le GPU via WebGL — des millions de cellules en temps réel.

Alan Turing (1952)Gray-ScottWebGL computeMorphogenèse
ACO Ant Colony Optimization

Des milliers de fourmis partent de nids et cherchent des sources de nourriture. Deux couches de phéromones coexistent : phéromone-nid (déposée en cherchant, guide le retour) et phéromone-food (déposée en retournant, guide vers la nourriture).

Contrairement à Physarum, les fourmis ont un état mémoire (cherche / retourne) et un comportement d'inversion de direction à chaque source trouvée. Les chemins les plus courts se renforcent automatiquement — c'est le principe utilisé pour optimiser des réseaux réels (routage, logistique).

Souris : Food place une source, Nest place un nid, Erase supprime une source, Scatter disperse les fourmis proches.

StigmergieDeux phéromonesMachine à étatsOptimisation combinatoire
